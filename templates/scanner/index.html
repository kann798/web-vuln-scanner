Frequently Asked Questions

Only scan websites you own or have explicit written permission to test. For safe practice, use intentionally vulnerable sites like testphp.vulnweb.com or demo.testfire.net — public test targets designed for security tools.

The scanner uses a Random Forest classifier trained on 2,000+ synthetic vulnerability patterns. It extracts 30 features from each URL and page — including query parameters, HTML signals, response headers, and URL patterns — then predicts the most likely vulnerability class alongside traditional rule-based checks.

  • Critical (75–100) — Immediate action required. Vulnerabilities like SQL Injection that can lead to full compromise.
  • High (50–74) — Serious risk. XSS and missing security headers enabling significant attacks.
  • Medium (25–49) — Moderate risk. CSRF or open redirects requiring specific conditions to exploit.
  • Low (1–24) — Minor informational exposures with limited direct impact.
  • Info (0) — No significant vulnerabilities detected.

Most scans complete in 10–40 seconds depending on target response time and parameter count. Pages with many form inputs take longer as each parameter is actively tested with multiple payloads. A 15-second per-request timeout prevents hanging.

No automated scanner is 100% accurate. VulnScanner AI may produce false positives or miss real vulnerabilities. It is a first-pass tool to surface likely issues quickly. Always validate critical findings manually or with a qualified security professional before remediation.

After a scan completes, a ⬇ Download PDF button appears in the results. You can also access reports from the History page. Reports include executive summary, risk score, all findings with payloads and fix code, and security recommendations.

This category checks for: missing security headers (X-Frame-Options, CSP, HSTS, X-Content-Type-Options), hardcoded API keys or passwords in HTML source, server version disclosure, plain HTTP usage, and database connection strings exposed in page source.

VulnScanner AI works well as a first-pass reconnaissance tool for bug bounty programs where you have permission to test. Use scan results as starting points for deeper manual investigation. Always read the program scope carefully and stay strictly within authorized targets.
